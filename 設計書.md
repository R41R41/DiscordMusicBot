# 設計書：Discord Music Bot（Windows デスクトップアプリ / TypeScript + discord.js + Electron）

## 目的
- 自作曲BGMを **Discord VCで参加者とリアルタイム共有**する
- 配信者が **Web UIで選曲・プレイリスト管理・再生操作**できる
- OBSの音を拾わず、**Botが音源ファイルを直接再生**して安定運用する
- **非エンジニアでも使える**ように Electron でデスクトップアプリとして配布する

> 注意：Discord音声の仕様変更（DAVE/E2EE）でボイス接続方式に影響が出る可能性があるため、音声送出部分は差し替え可能にする（Player/DiscordAdapter分離）。

## 全体構成

### 3層アーキテクチャ

```
┌─────────────────────────────────────────────┐
│              Electron Shell                  │
│  ┌───────────────────────────────────────┐  │
│  │         Web UI (React + Vite)         │  │
│  │  ライブラリ | プレイリスト | キュー | 設定  │  │
│  └──────────────┬──────────┬─────────────┘  │
│           REST API    WebSocket              │
│  ┌──────────────┴──────────┴─────────────┐  │
│  │        Backend (Node.js/TS)           │  │
│  │                                       │  │
│  │  WebApiService ─→ PlayerService       │  │
│  │                     ├→ LibraryService  │  │
│  │                     ├→ PlaylistService │  │
│  │                     └→ DiscordAdapter  │  │
│  │                          ↓             │  │
│  │                    Discord VC 🔊       │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

- 1プロセス（Node/TS）で Discord Bot + Web API + WebSocket を同居
- Electron がバックエンドプロセスの起動管理とフロントエンド表示を担当
- Web UI は Electron の BrowserWindow 内で表示（localhost:3001）

### 主要コンポーネント
- **PlayerService**：再生状態・キュー・ループ等の中核（UI/コマンドはここを操作）
- **DiscordAdapter**：VC接続と音声送出（ffmpeg使用、差し替え可能な設計）
- **LibraryService**：音源フォルダ再帰スキャン、ffprobe で duration 取得、検索、Track生成
- **PlaylistService**：プレイリストCRUD、.m3u ファイル永続化、旧JSON形式からの自動移行
- **WebApiService**：REST API + Socket.IO イベント
- **ConfigService**：設定の読み書き（config.json、環境変数対応）

### Electron (electron/main.cjs)
- バックエンドプロセスの spawn と管理
- バックエンド準備完了のポーリング待機（/api/system/status）
- シングルインスタンスロック
- システムトレイ（開く / ログを開く / DevTools / 終了）
- ポート競合時の自動プロセス終了（Windows netstat + taskkill）
- ログファイル出力（AppData 配下）

## データ

### Track
- `id`: string（ファイルパスの MD5 ハッシュ）
- `title`: string（ファイル名から拡張子を除いたもの）
- `path`: string（絶対パス）
- `duration`: number（秒、ffprobe で取得）

### Playlist
- `name`: string
- `trackIds`: string[]

### PlayerState（UIに配る）
- `connection`: disconnected / connecting / connected
- `current`: Track | null
- `queue`: Track[]
- `paused`: boolean
- `loop`: off / one / all
- `shuffle`: boolean
- `volume`: number（0–100）
- `position`: number（再生位置、秒）
- `guildId`: string | null
- `channelId`: string | null

---

## データ保存

### 設定ファイル（data/ 配下）
- `config.json`：Discord Token、音源フォルダパス、Webポート
- `settings.json`：volume / loop / shuffle

### プレイリスト（music/playlists/ 配下）
- 各プレイリストを個別の `.m3u` ファイルとして保存
- 形式：`#EXTM3U` ヘッダ + trackId を1行ずつ
- 変更時に即座にファイル更新
- 旧 `data/playlists.json` からの自動移行機能あり（初回起動時に変換、元ファイルは .bak にリネーム）

### Electron アプリのデータパス
- `%APPDATA%/discord-music-bot/DiscordMusicBot/`
  - `data/` - 設定ファイル
  - `music/` - 音楽ファイル + playlists/

## API

### Discord
- `GET /api/discord/guilds` - サーバー一覧
- `GET /api/discord/guilds/:guildId/channels` - VCチャンネル一覧

### Library
- `GET /api/library?query=...` - 曲検索
- `POST /api/library/rescan` - 再スキャン

### Playlists
- `GET /api/playlists` - プレイリスト一覧
- `POST /api/playlists` `{ name }` - 新規作成
- `DELETE /api/playlists/:name` - 削除
- `GET /api/playlists/:name` - プレイリスト詳細（トラック情報含む）
- `POST /api/playlists/:name/tracks` `{ trackId }` or `{ trackIds }` - 曲追加（単体 / 一括）
- `DELETE /api/playlists/:name/tracks/:trackId` - 曲削除
- `PUT /api/playlists/:name/order` `{ trackIds }` - 並び替え

### Player
- `GET /api/player/state` - 再生状態
- `POST /api/player/join` `{ guildId, channelId }` - VC接続
- `POST /api/player/leave` - VC切断
- `POST /api/player/play` `{ trackId }` - 即再生
- `POST /api/player/playlist` `{ name, mode: 'replace'|'append' }` - プレイリスト再生
- `POST /api/player/pause` - 一時停止
- `POST /api/player/resume` - 再開
- `POST /api/player/skip` - 次へ
- `POST /api/player/stop` - 停止
- `POST /api/player/seek` `{ position }` - シーク
- `POST /api/player/settings` `{ volume?, loop?, shuffle? }` - 設定変更
- `POST /api/player/queue` `{ trackId }` or `{ trackIds }` - キュー追加（単体 / 一括）
- `POST /api/player/queue/next` `{ trackId }` or `{ trackIds }` - 次に再生（単体 / 一括）
- `DELETE /api/player/queue` - キュークリア
- `DELETE /api/player/queue/:index` - キューから一件削除
- `PUT /api/player/queue/reorder` `{ fromIndex, toIndex }` - キュー並び替え

### Settings
- `GET /api/settings` - アプリ設定取得
- `POST /api/settings` `{ discordToken?, musicFolder?, webPort? }` - アプリ設定保存

### System
- `GET /api/system/status` - システムステータス（接続状態、バージョン等）
- `POST /api/system/open-folder` - 音楽フォルダをエクスプローラーで開く

### WebSocket（Push）
- `player_state`：状態変化時にブロードキャスト（UIはこれで同期）
- `library_updated`：ライブラリ再スキャン完了時

## 再生ルール
- `loop=one`：同じ曲を繰り返す
- `loop=all`：キュー末尾まで行ったら先頭に戻る（シャッフル時は再シャッフル）
- `loop=off`：キューが尽きたら停止
- 再生失敗（ファイル消失など）は「警告してスキップ」
- VC切断時はプレイヤーを停止（クラッシュ防止）
- シーク：ffmpeg で指定位置からデコードし直す方式

## ビルド & 配布

### 開発モード
- `npm run dev` - バックエンド（tsx watch）
- `npm run dev:web` - フロントエンド（Vite dev server）
- `npm run dev:electron` - Electron（開発モード）

### 本番ビルド
- `npm run dist:win` - Windows インストーラー生成
  1. `release/` フォルダをクリーンアップ
  2. TypeScript コンパイル
  3. Vite フロントエンドビルド
  4. electron-builder で NSIS インストーラー生成

### インストーラー設定
- ワンクリックインストール（NSIS oneClick）
- デスクトップ & スタートメニューにショートカット作成
- 上書きインストール対応（アンインストール不要）
