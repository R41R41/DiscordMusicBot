# 設計書：BGM共有 Discord Bot + Web UI（Windowsローカル / TypeScript + discord.js）

## 目的
- 自作曲BGMを **Discord VCで参加者とリアルタイム共有**する
- 配信者が **Web UIで選曲・プレイリスト管理・再生操作**できる
- OBSの音を拾わず、**Botが音源ファイルを直接再生**して安定運用する

> 注意：Discord音声の仕様変更（DAVE/E2EE）でボイス接続方式に影響が出る可能性があるため、音声送出部分は差し替え可能にする（Player/DiscordAdapter分離）。

---

## スコープ（v1）
### できること
- VC接続 / 切断
- ローカル音源再生（mp3/wav/flac）
- キュー再生（追加 / クリア / 表示）
- 再生操作（再生 / 一時停止 / 再開 / 次へ / 停止）
- ループ（off / one / all）
- シャッフル（on/off）
- 音量（0–100）
- 音源ライブラリ：指定フォルダをスキャン・検索
- プレイリスト：作成 / 削除 / 曲追加 / 曲削除 / 並び替え / 再生開始
- Web UI：再生パネル / ライブラリ / プレイリスト

### v1ではやらない（後回し）
- 曲のタグ読み（ID3等）・アートワーク
- フェード（曲切替フェード）
- 認証（Web UIを外部公開）
- 複数サーバー/複数VCの同時運用

---

## 想定利用
- 流す音：自作曲（ローカルファイル）
- 参加者：同じDiscord VCに参加
- 環境：配信PC（Windows）でBotをローカル実行
- サーバー：Level2（256kbps想定）※VCのチャンネル設定でビットレートを上げる

---

## 全体構成
- 1プロセス（Node/TS）で動かす
  - Discord Bot（VC接続・再生）
  - Web API（HTTP）
  - 状態Push（WebSocket）
- Web UIは `localhost` に提供（同一PCのみ）

### 主要コンポーネント
- **PlayerService**：再生状態・キュー・ループ等の中核（UI/コマンドはここを操作）
- **DiscordAdapter**：VC接続と音声送出（後で差し替えやすくする）
- **LibraryService**：音源フォルダスキャン、検索、Track生成
- **PlaylistService**：プレイリストCRUD、並び替え、再生投入
- **WebApiService**：REST API + WSイベント

---

## データ（ざっくり）
### Track
- `id`: 文字列（パス等から生成した安定ID）
- `title`: 文字列（初期はファイル名）
- `path`: 絶対パス

### Playlist
- `name`: 文字列
- `trackIds`: `string[]`

### PlayerState（UIに配る）
- `connection`: disconnected/connecting/connected
- `current`: Track | null
- `queue`: Track[]
- `paused`: boolean
- `loop`: off/one/all
- `shuffle`: boolean
- `volume`: number

## Web UI（画面）
### 1) Now Playing（再生パネル）
- 現在曲表示
- 再生/一時停止/次/停止
- ループ切替（off/one/all）
- シャッフルON/OFF
- 音量スライダー
- 接続状態表示

### 2) Library（曲一覧）
- 検索
- 「キューに追加」「今すぐ再生」
- 再スキャン

### 3) Playlists
- プレイリスト一覧（作成/削除）
- 中身の曲一覧（並び替え）
- 「このプレイリストを再生（replace/append）」
- 「ループ(all)で再生」ボタン（配信用プリセット）

---

## API（簡易）
### Library
- `GET /api/library?query=...`
- `POST /api/library/rescan`

### Playlists
- `GET /api/playlists`
- `POST /api/playlists` `{ name }`
- `DELETE /api/playlists/:name`
- `POST /api/playlists/:name/tracks` `{ trackId }`
- `DELETE /api/playlists/:name/tracks/:trackId`
- `PUT /api/playlists/:name/order` `{ trackIds }`

### Player
- `POST /api/player/join` `{ guildId, channelId }`
- `POST /api/player/leave`
- `POST /api/player/play` `{ trackId }`
- `POST /api/player/playlist` `{ name, mode: 'replace'|'append' }`
- `POST /api/player/pause`
- `POST /api/player/resume`
- `POST /api/player/skip`
- `POST /api/player/stop`
- `POST /api/player/settings` `{ volume?, loop?, shuffle? }`
- `GET /api/player/state`

### WS（Push）
- `player_state`：状態変化時に送る（UIはこれで同期）

---

## 再生ルール（要点だけ）
- `loop=one`：同じ曲を繰り返す
- `loop=all`：キュー末尾まで行ったら先頭に戻る（必要ならシャッフル）
- `loop=off`：キューが尽きたら停止
- 再生失敗（ファイル消失など）は「警告してスキップ」
